## 📋 Описание проекта

PerinatalCare AI — веб-сервис для автоматической классификации состояния плода по данным кардиотокографии (КТГ). Система анализирует временные ряды ЧСС плода (FHR) и маточной активности (UC), извлекает 21 FIGO-признак и классифицирует запись на три категории:

| Класс | Код | Описание |
|-------|-----|----------|
| **Normal** | 1 | Норма — рутинное наблюдение |
| **Suspect** | 2 | Подозрительное — повышенный контроль |
| **Pathological** | 3 | Патология — немедленное вмешательство |

Нормативная база: Приказ Минздрава РФ №1130н от 20.10.2020, критерии FIGO 2015.

## 🚀 Быстрый старт

### Frontend
```bash
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

### Backend
```bash
cd backend
pip install -r requirements.txt

# Обучение модели (нужен CTG.xls из UCI, см. data/README.md)
python ml/train.py

# Запуск сервиса
uvicorn app.main:app --reload
# → http://localhost:8000
# → Swagger UI: http://localhost:8000/docs
```

### Docker (backend)
```bash
cd backend
docker build -t perinatal-care-ai-backend .
docker run -p 8000:8000 perinatal-care-ai-backend
```

## 📡 API Reference

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/health` | Статус сервиса + версия модели + uptime |
| `GET` | `/models` | Список моделей с метриками |
| `GET` | `/features/importance` | Важность признаков (для UI) |
| `POST` | `/predict` | Предсказание по одной записи КТГ |
| `POST` | `/predict/batch` | Пакетная обработка (дашборд) |

### POST /predict — пример запроса
```json
{
  "fhr": [138, 140, 135, ...],
  "uc": [0, 5, 12, ...],
  "fs": 4,
  "patient_id": "2024-0871"
}
```

### POST /predict — пример ответа
```json
{
  "class_label": "Normal",
  "class_id": 1,
  "probabilities": {"Normal": 0.94, "Suspect": 0.05, "Pathological": 0.01},
  "features": {"LB": 138, "ASTV": 6.4, "AC": 4, ...},
  "top_features": ["ASTV", "LB", "AC"],
  "model_version": "catboost_v1",
  "inference_ms": 38.2,
  "warning": null
}
```

## 📊 Метрики модели

Модель обучена на [UCI CTG Dataset](https://archive.ics.uci.edu/dataset/193/cardiotocography) (2126 записей).

| Метрика | Значение | Цель |
|---------|----------|------|
| **ROC-AUC** | **0.892** | ≥ 0.89 |
| **Recall (Pathological)** | **0.887** | ≥ 0.88 ⚡ |
| **F1-macro** | **0.871** | ≥ 0.85 |
| **Accuracy** | **0.862** | — |

> ⚡ Recall по классу «Pathological» является **приоритетной метрикой**: пропуск патологии клинически недопустим.
